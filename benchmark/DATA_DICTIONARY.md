# Paperlight Benchmark Data Dictionary

The unit of analysis is one frozen text sample. `sampleId` is pseudonymous and carries no person, school, course, or assignment identity. The JSONL manifest contains metadata; the text body is a separate UTF-8 file whose canonical LF-normalized SHA-256 is recorded in `body.sha256`.

## Ground-truth labels

- `human_independent`: drafted without generative-AI text; spelling or reference-manager tools must be disclosed but do not change this label.
- `ai_generated`: the submitted prose was generated by one or more named model versions and received only mechanical formatting.
- `ai_draft_human_deep_edit`: AI supplied the draft and a human made substantive sentence-, argument-, and structure-level edits. The revision record determines `aiFraction` and spans.
- `human_draft_ai_polish`: a human supplied the draft and AI changed expression or organization. The exact accepted AI changes determine spans.
- `human_translated` / `ai_translated`: translation mechanism is recorded separately because translation can alter detector signals without changing the original author's reasoning.
- `documentClass` is `human`, `ai`, or `hybrid`. It is derived from the recorded production process, never from a detector score.
- `aiFraction` is the fraction of final canonical characters whose wording came from accepted generative-AI output. It is `0` for verified human text and `1` for fully generated text.
- `aiSpans` are half-open `[start,end)` offsets over canonical LF-normalized text. Two curators adjudicate hybrid spans from revision histories; unresolved cases are quarantined.

## Profile and features

`discipline`, `level`, `writingAbility`, `nativeEnglish`, `lengthBand`, and `textType` are grouping variables, not labels. Feature flags mark formulas, citations, tables, abbreviations, specialist terms, and template-driven prose so error concentrations are visible.

## Provenance and governance

`sourceType`, `sourceReference`, `authorization`, model versions, prompt summary, editing method, and `labelBasis` make each label auditable. `claimEligible=false` excludes development fixtures from product claims. `providerAccess` and `rewriteModelAccess` enforce contamination boundaries. A sealed blind sample must always have `rewriteModelAccess=never`.

## Prediction records

A prediction is separate from the manifest and keyed by dataset version, run, sample, provider, and provider version. `score` is a 0–1 risk score; it is not ground truth. Failed calls retain `status=failed` and a sanitized `errorCode`, with no invented score. `isMock=true` excludes the run from production threshold recommendations.
